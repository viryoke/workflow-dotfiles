# Nix devShells Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create the nix flake with 8 devShells (python, nodejs, rust, go, cpp, java, cuda, ai) for x86_64-linux and aarch64-darwin.

**Architecture:** Single flake.nix at `/Users/viryoke/arch-config/nix/flake.nix` with each devShell in a separate file under `nix/devShells/`. Uses `mkShellNoCC` for shells without C compiler needs, `mkShell` only for C++ and CUDA shells. All devShells are minimal — only the tools specified in the spec. CUDA devShell is x86_64-linux only (no CUDA on Mac). AI devShell uses claude-code and aider-chat from nixpkgs (no external flake input needed). Flake supports `nix develop ~/arch-config/nix#python` from any directory.

**Tech Stack:** nix flakes, nixpkgs-unstable, mkShellNoCC/mkShell

### Task 1: Create flake.nix

**Files:**
- Create: `/Users/viryoke/arch-config/nix/flake.nix`

- [ ] **Step 1: Create flake.nix**

```nix
{
  description = "viryoke dev environments";

  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixpkgs-unstable";
  };

  outputs = { self, nixpkgs }:
    let
      forAllSystems = nixpkgs.lib.genAttrs [
        "x86_64-linux"
        "aarch64-darwin"
      ];
      pkgsFor = forAllSystems (system: nixpkgs.legacyPackages.${system});
    in
    {
      devShells = forAllSystems (system:
        let pkgs = pkgsFor.${system}; in
        {
          python = import ./devShells/python.nix { inherit pkgs; };
          nodejs = import ./devShells/nodejs.nix { inherit pkgs; };
          rust = import ./devShells/rust.nix { inherit pkgs; };
          go = import ./devShells/go.nix { inherit pkgs; };
          cpp = import ./devShells/cpp.nix { inherit pkgs; };
          java = import ./devShells/java.nix { inherit pkgs; };
          ai = import ./devShells/ai.nix { inherit pkgs; };
          cuda = import ./devShells/cuda.nix { pkgs = pkgsFor.x86_64-linux; };
        }
      );
    };
}
```

Notes:
- `forAllSystems` generates outputs for both `x86_64-linux` (CachyOS PC) and `aarch64-darwin` (Mac)
- `cuda` devShell always uses `x86_64-linux` pkgs regardless of current system — CUDA only exists on Linux
- Each devShell file receives `{ pkgs }` argument — simple, no extra inputs needed
- No `llm-agents` external input — claude-code and aider-chat are in nixpkgs already

### Task 8: Create CUDA devShell

**Files:**
- Create: `/Users/viryoke/arch-config/nix/devShells/cuda.nix`

- [ ] **Step 1: Create cuda.nix**

```nix
{ pkgs }:

pkgs.mkShell {
  name = "cuda";

  packages = with pkgs; [
    cudaPackages_12.cudatoolkit
    cudaPackages_12.cudnn
    cudaPackages_12.cuda_nvcc
    python313
    numpy
  ];

  env = {
    CUDA_PATH = "${pkgs.cudaPackages_12.cudatoolkit}";
    EXTRA_LDFLAGS = "-L${pkgs.cudaPackages_12.cudatoolkit}/lib -L${pkgs.cudaPackages_12.cudnn}/lib";
    EXTRA_CFLAGS = "-I${pkgs.cudaPackages_12.cudatoolkit}/include -I${pkgs.cudaPackages_12.cudnn}/include";
  };

  shellHook = ''
    echo "CUDA ${pkgs.cudaPackages_12.cudaVersion} + cuDNN ${pkgs.cudaPackages_12.cudnn.version}"
  '';
}
```

Notes:
- `mkShell` (NOT mkShellNoCC) — CUDA needs stdenv for compilation
- `cudaPackages_12` — CUDA 12.9 toolkit (latest in nixpkgs)
- `cudatoolkit` — CUDA compiler and runtime libraries
- `cudnn` — Deep Neural Network library
- `cuda_nvcc` — NVIDIA CUDA compiler
- `python313` + `numpy` — for running/testing CUDA Python code
- `env` — sets `CUDA_PATH`, `EXTRA_LDFLAGS`, `EXTRA_CFLAGS` for proper linking
- **IMPORTANT**: This devShell is x86_64-linux only. On aarch64-darwin, `nix develop .#cuda` will fail because CUDA packages don't exist for Mac. This is expected and documented.
